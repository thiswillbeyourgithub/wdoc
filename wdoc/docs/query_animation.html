wdoc RAG Query Algorithm Flow

1
Embedding Creation

Text is split into chunks and embeddings are created for each chunk

↓
2
Query Expansion

User query is embedded and LLM generates alternative queries

↓
3
Embedding Search

Query embeddings used to find relevant documents (top 200)

↓
4
Document Filtering

Small LLM evaluates if each document is relevant to query

↓
5
Iterative Search

If >90% documents are relevant, search with higher top_k

↓
6
Information Extraction

Strong LLM extracts relevant info from documents

↓
7
Semantic Batching

Group similar answers using hierarchical clustering

↓
8
Answer Combination

LLM combines each batch into single answer recursively

↓
9
Final Answer

Coherent response with references to source documents
